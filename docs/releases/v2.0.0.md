# Cortex v2.0.0 Release Notes

Date: 2026-07-10
Tag: `v2.0.0`

`v2.0.0` makes Cortex a *governed* memory primitive: memory splits into an explicit layer you author and an implicit layer your agents record â€” and the only bridge between them is your explicit accept. The positioning in one line: **the memory primitive that proposes, never writes.**

Backward-compatible at the database layer: every schema change is a new additive table, and your existing `~/.cortex` upgrades in place on first run.

## Shipped in v2.0.0 (merged on `main`)

### Directives â€” the explicit layer
- **New `directives` table** (additive migration): human-authored rules ("always X", "never Y") as a first-class memory kind.
- **Pinned retrieval**: active directives always surface above BM25/semantic results, flagged `kind:"directive"`, without consuming the requested result budget. They can never lose to ranking noise.
- **Never decay, no vectors**: directives have no decay rate and are deliberately excluded from the embedding path â€” exact/FTS + pinning only.
- **CLI**: `cortex directive add|list|edit|archive|rm`. **MCP**: `cortex_directive_add`, `cortex_directive_list`.

### Session ledger â€” the implicit layer
- **New append-only `session_ledger` table**: agents record task outcomes (summary, `success|partial|failure`, files touched, fix pattern) at end-of-task. No update or delete path exists anywhere in the store.
- **CLI**: `cortex ledger record|list --since`. **MCP**: `cortex_ledger_record` â€” the end-of-task call for any MCP-connected agent.

### The proposer loop â€” propose, never write
- **`cortex propose scan [--min-occurrences 3] [--window 14d] [--dry-run]`**: groups recurring ledger fix patterns by exact match â€” deterministic, no LLM, no network â€” and files *pending proposals* carrying their evidence (the ledger rows that triggered them).
- **`cortex propose accept <id>`** is the only code path in the binary that turns a proposal into a directive. `dismiss` closes it at zero cost. The scan-writes-nothing guarantee is enforced by test (`TestPropose_GovernanceGuarantee`), not convention.
- **MCP**: `cortex_propose_list` (read-only). Accept and dismiss are deliberately not exposed over MCP â€” the write stays human-gated at the CLI.

### Titled citations
- Every answer/search citation now carries a human-readable `title` â€” derived from the section header, else the base filename, else (for directives) the rule text â€” instead of a bare `file:line` locator. Pure derivation; no LLM.

### Hygiene + repositioning
- OpenClaw plugin demoted from core to `examples/openclaw-plugin/` as a reference adapter; the core reads runtime-neutral.
- CI: `concurrency: cancel-in-progress` on all workflows, daily canaries demoted to weekly, and the metered-LLM quality eval is dispatch-only â€” nothing spends money on a schedule.
- README repositioned around the two-layer memory + the propose-never-write loop.

## Upgrade notes

- No action required. First run after upgrade applies the additive migrations (`directives`, `session_ledger`, `directive_proposals`) behind idempotent meta flags.
- New MCP tool count: 21. Existing tool names and schemas are unchanged.
