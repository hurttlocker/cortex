# Cortex v0.8.0 Release Notes

Date: 2026-02-25
Tag: `v0.8.0`

## Summary

`v0.8.0` is the **Knowledge Graph Intelligence** release. Five major features turn Cortex from a storage+search tool into an analytical knowledge platform: RRF search fusion, impact analysis, topic clustering, graph-aware MCP tools, and temporal timeline views. Inspired by GitNexus patterns, adapted for personal knowledge graphs.

## Highlights

### üîÄ RRF Search Mode (#205)
- **Reciprocal Rank Fusion** combines BM25 keyword and semantic vector results using rank-based scoring
- Configurable K parameter (default 60), parallel execution of both search backends
- Graceful fallback when embedder unavailable ‚Äî degrades to BM25 only
- `cortex search "query" --mode rrf`

### üí• Impact / Blast-Radius Analysis (#206)
- New endpoint: `GET /api/impact?subject=X&depth=N&min_confidence=0.5`
- Groups related facts by semantic predicate categories (has_tool, depends_on, has_config, etc.)
- Returns confidence distribution (high/medium/low), connected subjects, and traversal depth
- Visualizer adds concentric ring layout with confidence heat coloring

### üèò Topic Clustering (#207)
- Subject co-occurrence analysis builds topic communities at import time
- Algorithm: co-occurrence edges ‚Üí connected components ‚Üí merge small clusters ‚Üí cohesion scoring
- Incremental updates: 10% novel subject threshold triggers full rebuild, otherwise assigns to existing clusters
- New CLI: `cortex cluster [--rebuild] [--name "trading"] [--export json]`
- New API: `GET /api/clusters`, `GET /api/clusters/:id`
- Visualizer sidebar with cohesion badges, drill-down navigation

### üß∞ Graph-Aware MCP Tools (#208)
- **3 new tools:** `graph_explore`, `graph_impact`, `list_clusters`
- **2 new resources:** `cortex://graph/subjects`, `cortex://graph/clusters`
- Source prefix filtering on graph_explore (useful with connectors)
- Graceful cluster table detection ‚Äî no crash on pre-migration DBs
- **Total: 19 MCP tools, 4 resources**

### ‚è≥ Temporal Flow Timeline (#209)
- New endpoint: `GET /api/timeline?subject=X&from=2026-01-01&to=2026-02-25&bucket=day`
- Day/week/month bucketing with fact aggregation
- Transition detection: superseded, refined, decayed, expanded, contradicted
- Confidence trend tracking across buckets
- Related subject expansion via co-occurrence and edge traversal
- Full visualizer integration with dedicated timeline mode

## Schema Changes

- New tables: `clusters`, `fact_clusters` (auto-created via migration)
- `ExportNode` struct gains `cluster_id`, `cluster_color`, `depth` fields
- No breaking changes to existing tables or APIs
